# 网络剪贴板共享工具

多台设备在局域网内共享剪贴板文本内容。A 设备发送 → B 设备实时同步，支持历史记录、删除、自动推送刷新。

## 功能特性

| 功能 | 说明 |
|------|------|
| 文本共享 | 客户端推送文本到服务端，其他客户端可拉取 |
| 历史记录 | 服务端保存最近 10 条剪贴板，GUI 客户端表格展示 |
| 一键复制 | 点击历史记录中的"复制"按钮，文本直接写入系统剪贴板 |
| 删除管理 | 用户可删除自己发送的历史记录 |
| 自动同步 | 服务端在 PUSH/DELETE 操作后主动推送刷新通知，所有客户端实时更新 |
| 重复检测 | 发送文本前检查历史记录中是否已存在相同内容，避免重复 |
| 多用户并发 | 多线程架构，每个客户端连接独立线程处理，互不干扰 |
| GUI 客户端 | 图形化界面，三列表格（用户名 / 内容 / 操作），操作直观 |
| CLI 客户端 | 命令行客户端，适合脚本集成和远程终端使用 |

## 快速开始

### 环境要求

- JDK 8 或以上
- 无第三方依赖，纯 Java 标准库

### 编译

```bash
javac -encoding UTF-8 -d out -sourcepath src src/com/clipboard/util/SimpleLogger.java src/com/clipboard/protocol/Protocol.java src/com/clipboard/server/*.java src/com/clipboard/client/*.java
```

### 启动服务端

```bash
java -cp out com.clipboard.server.ClipboardServer
```

默认监听端口 `8888`，也可指定端口：

```bash
java -cp out com.clipboard.server.ClipboardServer 9999
```

### 启动 GUI 客户端

```bash
java -cp out com.clipboard.client.GUIClipboardClient
```

### 启动 CLI 客户端

```bash
java -cp out com.clipboard.client.ClipboardClient 127.0.0.1 8888
```

## 服务端配置与使用

### 启动

```bash
# 默认端口 8888
java -cp out com.clipboard.server.ClipboardServer

# 自定义端口
java -cp out com.clipboard.server.ClipboardServer 9999
```

### 启动日志

```
[Server] Listening on port 8888...
[Server] New connection from /127.0.0.1:54321
[Server] PUSH from /127.0.0.1:54321, length=11
[Server] Broadcasted NOTIFY_REFRESH to all clients after PUSH from /127.0.0.1:54321
[Server] PULL from /127.0.0.1:54322
[Server] Connection closed: /127.0.0.1:54321
```

### 服务端核心功能

| 功能 | 说明 |
|------|------|
| 存储剪贴板 | 内存中维护最近 10 条历史记录 |
| 处理 PUSH | 接收客户端推送的文本，追加到历史 |
| 处理 PULL | 返回最新一条文本 |
| 处理 HISTORY | 返回全部历史记录（JSON 格式） |
| 处理 DELETE | 删除指定索引的历史记录 |
| 广播通知 | PUSH/DELETE 成功后推送 `NOTIFY_REFRESH` 给所有在线客户端 |
| 日志输出 | 记录所有连接、操作、广播事件 |

## 客户端使用

### GUI 客户端

启动后界面分为三个区域：

```
┌──────────────────────────────────────────────────┐
│  连接设置                                         │
│  主机: [127.0.0.1]  端口: [8888]  用户名: [___]  │
│  [连接]  [断开]  ● 已连接                        │
├──────────────────────────────────────────────────┤
│  共享剪贴板历史                        [刷新历史]  │
│  ┌──────┬─────────────────────┬────────────┐     │
│  │用户名│ 内容                 │ 操作        │     │
│  ├──────┼─────────────────────┼────────────┤     │
│  │ 小明 │ Hello World!        │ [复制]     │     │
│  │ 小红 │ 这是一段长文本...    │ [复制][删除]│    │
│  └──────┴─────────────────────┴────────────┘     │
├──────────────────────────────────────────────────┤
│  发送文本                                         │
│  ┌──────────────────────────────────────────┐    │
│  │ 输入要发送的文本...                        │    │
│  └──────────────────────────────────────────┘    │
│  [发送到共享剪贴板]                                │
└──────────────────────────────────────────────────┘
```

#### 操作步骤

1. **连接**：输入主机地址（默认 `127.0.0.1`）、端口（默认 `8888`）、用户名，点击"连接"
2. **发送文本**：在下方的文本框中输入内容，点击"发送到共享剪贴板"（或按 Enter）
3. **复制文本**：点击历史记录中任意条目旁的"复制"按钮，文本自动写入系统剪贴板
4. **删除记录**：点击自己发送的条目旁的"删除"按钮（需确认）
5. **刷新历史**：点击"刷新历史"按钮手动拉取最新数据
6. **断开连接**：点击"断开"按钮

> **自动同步**：当其他客户端发送或删除内容时，历史记录会自动刷新，无需手动操作。

#### 重复检测

发送文本时，客户端会先拉取服务端最新历史记录，检查是否存在相同内容。如果存在，会弹出提示：

```
检测到重复内容：该文本已在历史记录中存在，是否仍要发送？
```

### CLI 命令行客户端

```bash
java -cp out com.clipboard.client.ClipboardClient 127.0.0.1 8888
```

#### 交互命令

```
> push
Enter text to push: Hello from device A
[Client] PUSH success, 17 bytes sent

> pull
[Client] Received: Hello from device A

> exit
[Client] Disconnected
```

| 命令 | 说明 |
|------|------|
| `push` | 输入文本并推送到服务端 |
| `pull` | 从服务端拉取最新文本 |
| `exit` | 断开连接并退出 |

## 应用层协议

### 消息格式（定长头 + 变长体）

```
┌──────────┬──────────────┬──────────────────┐
│ 命令字   │  数据长度     │  数据内容         │
│ 1 byte  │  4 bytes(大端) │  N bytes(UTF-8)  │
└──────────┴──────────────┴──────────────────┘
```

| 字段 | 长度 | 类型 | 说明 |
|------|------|------|------|
| 命令字 | 1 byte | int | 操作类型标识 |
| 数据长度 | 4 bytes | int (大端序) | 数据内容的字节数，可为 0 |
| 数据内容 | N bytes | byte[] | UTF-8 编码的文本 |

### 命令定义

| 命令字 | 名称 | 方向 | 说明 |
|--------|------|------|------|
| 1 | PUSH | 客户端 → 服务端 | 推送文本到服务端 |
| 2 | PULL | 客户端 → 服务端 | 从服务端拉取最新文本 |
| 3 | OK | 服务端 → 客户端 | 操作成功，数据内容为返回的文本 |
| 4 | ERROR | 服务端 → 客户端 | 操作失败，数据内容为错误信息 |
| 5 | HISTORY | 客户端 → 服务端 | 请求全部历史记录 |
| 6 | DELETE | 客户端 → 服务端 | 删除指定索引的历史记录 |
| 7 | NOTIFY_REFRESH | 服务端 → 客户端 | 通知客户端刷新历史（服务端推送） |

### 交互流程示例

**PUSH 流程：**

```
客户端                                   服务端
  │   命令字=1  长度=11  "hello world"      │
  │──────────────────────────────────────►│
  │                          更新历史       │
  │   命令字=3  长度=0  (OK)               │
  │◄──────────────────────────────────────│
  │                    广播 NOTIFY_REFRESH  │
  │◄──(其他客户端收到通知，自动刷新)───────│
```

**PULL 流程：**

```
客户端                                   服务端
  │   命令字=2  长度=0                     │
  │──────────────────────────────────────►│
  │                          读取最新文本   │
  │   命令字=3  长度=11  "hello world"      │
  │◄──────────────────────────────────────│
```

## 设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| **策略模式** | `CommandHandler` 接口 + 各命令处理器 | 不同命令有不同的处理逻辑，可扩展 |
| **工厂模式** | `CommandHandler` 工厂注册 | 通过命令字映射到对应的处理器 |
| **观察者模式** | `ClientManager` 广播机制 | 服务端维护客户端列表，数据变更时主动推送 `NOTIFY_REFRESH` |
| **单例模式** | `ClipboardHistoryManager` | 历史记录管理器全局唯一实例 |
