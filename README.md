# 网络剪贴板共享工具

多台设备在局域网内共享剪贴板文本内容。A 设备复制 → B 设备粘贴。

## 使用场景

- 同一房间多人协作
- 办公多设备同步
- 跨设备（笔记本/台式机）复制粘贴文本

## 架构

C/S 模式（客户端-服务器），一个服务端 + 多个客户端。

## 技术栈

| 技术项     | 说明                              |
| ---------- | --------------------------------- |
| 编程语言   | Java（JDK 8+）                    |
| 通信协议   | TCP                               |
| 并发模型   | 多线程（每个客户端连接一个线程）  |
| 应用层协议 | 自定义（定长消息头 + 变长消息体） |
| 第三方依赖 | 无（纯 Java 标准库）              |

## 快速开始

### 编译

```bash
javac -d out src/**/*.java
```

### 启动服务端

```bash
java -cp out server.Server
```

默认监听端口 `8888`。

### 启动客户端

```bash
java -cp out client.Client <服务端IP> <端口>
```

示例：

```bash
java -cp out client.Client 192.168.1.100 8888
```

### 客户端命令

| 命令   | 说明                       |
| ------ | -------------------------- |
| `push` | 输入文本并推送到服务端     |
| `pull` | 从服务端拉取最新文本       |
| `exit` | 断开连接并退出             |

## 应用层协议

```
┌──────────┬──────────────┬──────────────────┐
│ 命令字   │  数据长度     │  数据内容         │
│ 1 byte  │  4 bytes(大端) │  N bytes(UTF-8)  │
└──────────┴──────────────┴──────────────────┘
```

| 命令字 | 名称  | 方向            | 说明                           |
| ------ | ----- | --------------- | ------------------------------ |
| 1      | PUSH  | 客户端 → 服务端 | 推送文本到服务端               |
| 2      | PULL  | 客户端 → 服务端 | 从服务端拉取最新文本           |
| 3      | OK    | 服务端 → 客户端 | 操作成功，数据内容为返回的文本 |
| 4      | ERROR | 服务端 → 客户端 | 操作失败，数据内容为错误信息   |
